# Narrative System (v1.0 — approved copy source)

## Spine
One trait explains everything: **he has to understand the whole system, not just make it work.**
- Understanding → quality (you can't predict failure in a system you don't understand)
- Understanding → automation (you can't eliminate a workflow you haven't mapped)
- Understanding → AI (his FYP combines an LLM with a trained model — systems thinking applied to intelligence itself)

Never state the spine as a slogan. It's shown through fragments; each section carries one facet.

## Voice rules
- First person, plain sentences, concrete nouns. No "passionate," "journey," "crafting digital experiences."
- Metrics where true; stories where metrics don't exist.
- Every fragment ≤ 3 sentences except About.
